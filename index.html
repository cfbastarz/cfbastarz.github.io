catastrophic error: cannot open source file "malloc.h"

I got this error when trying to compile the bacio library and the compiler coudn't open the malloc.h header. This happen on OS X Mountain Lion:

bacio.v1.3.c(14): catastrophic error: cannot open source file "malloc.h"
  #include 
         ^

compilation aborted for bacio.v1.3.c (code 4)
make[1]: *** [bacio.v1.3.o] Error 4
make: *** [linux] Error 2

Searching on the internet, I figured out a solution and it is a modification on a routine, which worked perfectly for my case. All you need to do is open the problematic routine source file (in my case "bacio.v1.3.c"), search for the line with "malloc.h" and make the following modification:

#if !defined(__APPLE__)
#include 
#endif

By doing that the malloc.h header can be find on the OS X Mountain Lion!
